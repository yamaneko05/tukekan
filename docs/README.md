# 友人間貸し借り管理アプリ - 要件定義書

## 概要

友人間のお金の貸し借りを管理するWebアプリケーション。
スマホからの入力のしやすさと、貸借状況の可視化を重視する。

## 用語定義

| 用語          | 定義                                                     |
| ------------- | -------------------------------------------------------- |
| Account       | アプリに登録した認証ユーザー                             |
| Partner       | 貸借相手（アプリ未登録の友人も含む）                     |
| Transaction   | 1件の貸し借り記録                                        |
| 貸し          | 自分が相手にお金を渡した（+金額）                        |
| 借り/返済     | 自分が相手からお金を受け取った（-金額）                  |
| linkedAccount | Partnerに紐付いた、アプリ登録済みのAccount               |
| Group         | （将来）ユーザーをまとめる単位、グループ内のみデータ共有 |

## バイブコーディング時の参照ガイド

### 基本的な進め方

1. `07-phases.md` のタスクを1つ選ぶ
2. 関連ドキュメントを明示的に渡す
3. 具体的な指示を出す
4. 動作確認 → 完了したら `[x]` に更新 → 次のタスクへ

### プロンプト例

```
以下のドキュメントを参照して実装してください:
- docs/05-tech-stack.md（技術スタック・ディレクトリ構成）
- docs/04-screens.md（画面設計）

【タスク】Phase 1-1: 基本レイアウト（ヘッダー、ボトムバー、FAB）を実装

【要件】
- ヘッダー: 右に設定アイコン
- ボトムバー: ホーム、メンバーの2項目
- FAB: 右下固定、+アイコン
```

### タスク別の参照ドキュメント

| タスク種別       | 参照ドキュメント                      |
| ---------------- | ------------------------------------- |
| 画面実装         | `04-screens.md` + `05-tech-stack.md`  |
| DB操作           | `03-data-design.md`                   |
| 認証実装         | `06-security.md` + `05-tech-stack.md` |
| 機能確認         | `02-features.md`                      |
| ディレクトリ確認 | `05-tech-stack.md`                    |
| フェーズ進捗     | `07-phases.md`                        |

### ポイント

- 1回の指示で1タスクに絞る
- 関連ドキュメントは毎回明示的に指定する
- `07-phases.md` をチェックリストとして使う
